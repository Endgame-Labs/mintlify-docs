---
title: "Querying Endgame via URL"
description: "You can launch Endgame queries directly from a URL—useful for bookmarks, Salesforce and Slack workflows, or embedding links in dashboards and documentation."
---

### Base URL

```
https://app.endgame.io/threads/new
```

### Query Parameters

| Parameter      | Description                                                                    | Example                                     |
| -------------- | ------------------------------------------------------------------------------ | ------------------------------------------- |
| `firstMessage` | The prompt you want Endgame to answer (URL-encoded)                            | `firstMessage=what's+the+latest+in+my+book` |
| `template_id`  | The ID of a saved Endgame template to run—use this _instead of_ `firstMessage` | `template_id=abc123`                        |
| `accountId`    | The Salesforce Account ID you want the thread scoped to                        | `accountId=0015f00000ABC123`                |

To find template IDs, Admins can navigate to https://app.endgame.io/settings/templates, click the dropdown menu on any template, and select **Copy ID**.

### Examples

**Free-form prompt:**

```
https://app.endgame.io/threads/new?firstMessage=summarize+my+recent+action+items
```

**Using a template:**

```
https://app.endgame.io/threads/new?template_id=abc123&accountId=0015f00000ABC123&mode=research
```

### A note on URL encoding

Prompts need to be URL-encoded (e.g., spaces become `+` or `%20`). This is standard web behavior, not Endgame-specific. See [MDN's guide to URL encoding](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/encodeURIComponent) for more detail.
