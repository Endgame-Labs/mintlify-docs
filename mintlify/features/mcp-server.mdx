## What is the Endgame MCP Server?

The Endgame MCP (Model Context Protocol) Server allows AI assistants like Claude to directly communicate with Endgame. Once connected, your assistant can access Endgame tools to search interaction history, look for upcoming meetings, find relevant stakeholders and more.

## What You Can Do

The MCP server essentially gives your AI assistant a direct line to Endgame's knowledge base. Here are some examples of what you can do with the Endgame MCP Server connected to your AI assistant:

## What kinds of questions can Endgame answer?

- What accounts does my team own, and when was the last time we interacted with each one?

- What did customers bring up as pain points or risks across our last 20 sales calls?

- What did Salesforce's CFO say about their AI strategy on the most recent earnings call?

- Show me all open opportunities over $500K that haven't had any activity in the last 45 days.

- What are we saying internally on Slack about the upcoming renewal with Acme Corp?

## Getting Started

### Claude

1. An Admin needs to add Endgame as a custom connector before individuals on the team can connect to Endgame. Go to Customize → Connectors.
2. Click Add custom connector in the "+" menu.
3. Enter `https://app.endgame.io/api/v1/mcp` as the URL.
4. Click Add and follow the connection prompts to authenticate to Endgame.
5. Once connected at the organization level, individuals can now connect to Endgame as well.

   **Note**: If you had previously installed the legacy Endgame MCP, you may need to delete it before connection the new MCP or give the new one a different name to avoid conflicts.

    ### Step 1: Get Your API Key

    1. Log in to Endgame
    2. Navigate to **Settings** → **API Keys** - you will only be able to view your api keys, you won't see the other keys generated by people on your team
    3. Create a new API key
    4. Copy and save your token securely (you won't be able to see it again!)
    5. Provide the Organization ID available in the connection instructions when prompted

    Admins can create API keys for members of their organization at their discretion. We recommend using descriptive names like "sarah-endgame-key" to make it easy to identify and revoke keys when needed. Admins will need to copy the key and share it manually with the end user. Keys are tied to the creator, so queries that reference "me" or "my" will return results specific to the key creator. For example, "What is new on my accounts this week?" would reflect the admin's accounts, not the end user's.
